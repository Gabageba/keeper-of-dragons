# 33 — Тестирование, аналитика и релиз

**Фаза:** 8 — Мобилка и релиз · **Зависит от:** всё

## Цель

Довести до публикуемого состояния: ловить регрессии, понимать поведение игроков,
выпустить веб-версию и (опц.) сборки в сторы.

## Что делать

1. **Автотесты на «чистую» логику.** Юнит-тесты (Vitest) на детерминированные
   функции без Phaser: расчёт накопления ресурсов (task-06), оффлайн-начисление
   (task-04), бросок шанса скрещивания (task-11), множители бонусов (task-22),
   рост растений (task-08). Это самый дешёвый и ценный слой тестов для idle-игры.
2. **Smoke-тест сборки.** `npm run build` + ручной прогон ключевой петли: вырастить
   → покормить → собрать → скрестить → вылупить → сдать заказ → открыть остров.
   Чек-лист регрессий в `docs/qa-checklist.md`.
3. **Тест времени и оффлайна.** Чит-перемотка (task-26): таймеры завершаются,
   ресурсы начисляются, ничего не уходит в минус/NaN. Смена системного времени
   назад не ломает игру (clamp отрицательного `elapsed`).
4. **Тест на устройствах.** Несколько телефонов/браузеров (iOS Safari, Android
   Chrome): FPS, тач, безопасные зоны, аудио-автоплей.
5. **Аналитика (опц., приватно).** Лёгкие события: ретеншн, на каком виде/острове
   застревают, длина сессии. Без PII. Помогает балансу (task-26).
6. **Обработка ошибок.** Глобальный `window.onerror`/`unhandledrejection` → не
   крашить молча; защита сейва от порчи (бэкап-ключ, task-04).
7. **Релиз.**
   - Веб: `vite build` → статик-хостинг (GitHub Pages/Netlify/Vercel), `base:'./'`
     уже настроен. PWA-манифест (опц.) для «установки» с браузера.
   - Мобилки (опц.): сборка через Capacitor (task-31), подпись, стор-листинги,
     скриншоты, иконка.
8. **Версионирование.** Семвер, changelog, бамп `version` в сейве при изменении
   схемы (миграции task-04).

## Definition of Done

- [ ] Юнит-тесты покрывают ключевую idle-математику и зелёные в CI/локально.
- [ ] QA-чеклист основной петли проходит без регрессий.
- [ ] Перемотка времени/оффлайн не дают NaN/минусов/потерь прогресса.
- [ ] Игра протестирована минимум на iOS Safari и Android Chrome.
- [ ] Веб-версия задеплоена и запускается с чистого устройства.
- [ ] Ошибки логируются, сейв защищён от порчи.
