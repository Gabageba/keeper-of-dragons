# 31 — Сборка под мобилки через Capacitor

**Фаза:** 8 — Мобилка и релиз · **Зависит от:** 00, играбельная сборка

## Цель

Обернуть тот же Vite-билд в нативное приложение iOS/Android через Capacitor, без
переписывания. Веб-версия = тот же билд в браузере.

## Что делать

1. **Установка.** `@capacitor/core`, `@capacitor/cli`, `@capacitor/ios`,
   `@capacitor/android`. `npx cap init` → `webDir: 'dist'`, appId/appName.
   `vite.config` уже с `base: './'` (нужно для `file://`).
2. **Платформы.** `npx cap add ios`, `npx cap add android`. Сборка:
   `npm run build && npx cap sync`.
3. **Жизненный цикл приложения.** Через `@capacitor/app` слушать pause/resume →
   принудительный сейв (task-04) и пауза музыки (task-29). Корректное оффлайн-
   начисление при resume.
4. **Экран/вырез.** `@capacitor/status-bar`, безопасные зоны, ориентация (заблок.
   портрет или поддержать обе — решить). Splash screen (`@capacitor/splash-screen`).
5. **Хранилище.** LocalStorage работает в WebView, но для надёжности на мобилке —
   `@capacitor/preferences` или IndexedDB (стабильнее на iOS). Подменить `storage`
   у Zustand `persist` соответствующим адаптером (task-04).
6. **Производительность.** Проверить WebGL в WebView, FPS, лимиты частиц (task-28),
   размер атласов (память). Тест на реальном среднем устройстве.
7. **Иконки/сплэш.** Сгенерировать (`@capacitor/assets`) из исходников арта.

## Definition of Done

- [ ] `npx cap sync` собирает рабочее приложение на iOS и Android.
- [ ] Прогресс не теряется при сворачивании/закрытии (pause→save, resume→offline).
- [ ] UI учитывает вырезы/безопасные зоны; ориентация фиксирована/корректна.
- [ ] Стабильный FPS и приемлемая память на среднем телефоне.
- [ ] Иконка и сплэш-скрин на месте.
