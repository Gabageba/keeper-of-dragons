# 04 — Сохранение/загрузка и оффлайн-прогресс

**Фаза:** 0 — Фундамент · **Статус:** ✅ выполнено · **Зависит от:** 03

## Цель

Прогресс сохраняется локально (LocalStorage → позже IndexedDB), таймеры считаются
от системного времени (`Date.now()`), а при возвращении в игру корректно
начисляется накопленное за оффлайн время. Серверной валидации нет (пет-проект).

## Что сделано

Сохранение — на **Zustand `persist`**. Прежний ручной `SaveManager` и `AutoSave`
удалены: persist сам пишет в LocalStorage на каждое изменение состояния.

1. **Один блоб.** `useGameStore` обёрнут в `persist` с ключом `dragons-game`,
   формат `{ version, state }`. `partialize` сохраняет только данные (game/dragons/
   island/garden/breeding), исключая транзиентную `grid` и экшены. На каждой записи
   `last_save` штампуется текущим временем.
2. **Загрузка.** persist рехидрирует синхронно при импорте стора. `BootScene` после
   загрузки контента дёргает `onBoot` (мост): `setCurrentIsland` строит сетку,
   `applyOffline` начисляет оффлайн.
3. **Автосейв не нужен.** persist пишет синхронно на каждый `set`; отдельный таймер
   и обработчики `visibilitychange`/`pagehide`/`blur` больше не требуются.
4. **Оффлайн-начисление** (`store.applyOffline`): для каждого взрослого дракона
   `elapsed = now - last_collected` → ресурсы = `production/час × часы` (с учётом
   бафов биома и потолка склада); отмечается готовность скрещивания и яиц
   (`ready_at <= now`). Возвращает сводку → `useUIStore.offlineSummary` → React-окно
   «пока вас не было».
5. **Потолок склада.** `STORAGE_CAP` в `store/types.ts`; снимается, если есть взрослый
   дракон с `global_bonus === 'infinite_storage'` (Вечнозим). Баланс — task-26.
6. **Миграции версий.** `persist({ version: 1 })` + хук `migrate` — точка расширения.
   Старые сейвы (`dragons_save_v1`) намеренно не мигрируются (ранний проект).
7. **Сброс.** Dev-меню удаляет ключ `dragons-game` и перезагружает; кнопки симуляции
   («1ч оффлайн», «Завершить таймеры») правят persisted-блоб и перезагружают.
8. **Переход на IndexedDB позже** — заменить `storage` у persist на IndexedDB-адаптер
   (`idb-keyval`), форма блоба та же.

## Definition of Done

- [x] Перезагрузка страницы сохраняет прогресс (монеты, драконы, постройки).
- [x] Сворачивание/закрытие вкладки не теряет прогресс (persist пишет на каждый set).
- [x] После 1 часа оффлайна дракон 8/ч даёт ~8 ресурсов (в пределах потолка).
- [x] Таймеры скрещивания/инкубации завершаются «задним числом» (в сводке оффлайна).
- [x] Кнопка сброса сохранения доступна в dev-меню.
