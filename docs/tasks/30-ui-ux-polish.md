# 30 — UI/UX полировка и обучение

**Фаза:** 7 — Арт и полировка · **Зависит от:** все UI-сцены

## Цель

Привести интерфейс к единому стилю, сделать удобным на тач-экранах, добавить
онбординг (туториал) и обратную связь на каждое действие.

## Что делать

1. **Единый UI-кит.** Переиспользуемые компоненты (кнопка, панель, модалка,
   таб-бар, тост-уведомление, бейдж редкости, прогресс-кольцо таймера). Палитра/
   шрифты из стиля GDD (золото/тёмный фон, Cinzel-подобные шрифты). Складывать в
   `src/ui/`.
2. **Главный HUD (React, `ui/hud/`).** Валюты (монеты/камни), уровень+полоса опыта, кнопки:
   Книга, Заказы, Скрещивание, Инвентарь, Карта, Построить, Собрать всё, Настройки.
   Адаптив под портрет/ландшафт.
3. **Тач-эргономика.** Кнопки ≥44px, безопасные зоны (`viewport-fit=cover`, notch),
   крупные хит-зоны, без ховер-зависимостей, понятные жесты (drag-pan, pinch-zoom).
4. **Обратная связь.** Тосты («+150 монет», «Новый вид открыт!»), подтверждения на
   трату премиум-валюты, индикаторы «недоступно» с причиной (нет ресурсов/уровня/
   слота).
5. **Онбординг/туториал.** Завязан на сюжетные квесты (task-14): подсветка нужной
   кнопки, пошаговые подсказки на первых действиях (вырасти Эмберикса → построй сад
   → сделай скрещивание → открой второй остров). Пропускаемый.
6. **Индикаторы «есть действие».** Бейджи на кнопках (готовые яйца, выполнимые
   заказы, собираемые ресурсы) — игрок сразу видит, куда идти.
7. **Локализация-ready.** Все строки — через словарь (`ru` сейчас, структура под
   будущие языки), не хардкодить в сценах.
8. **Настройки.** Звук (task-29), язык, сброс прогресса (с подтверждением), о игре.

## Definition of Done

- [ ] Единый визуальный стиль во всех сценах через UI-кит.
- [ ] Удобно играть одной рукой на телефоне (портрет и ландшафт).
- [ ] Туториал проводит нового игрока через базовую петлю.
- [ ] Каждое действие даёт понятную обратную связь; недоступность объяснена.
- [ ] Бейджи-индикаторы показывают, где есть действие.
- [ ] Строки вынесены в словарь (локализация-ready).
