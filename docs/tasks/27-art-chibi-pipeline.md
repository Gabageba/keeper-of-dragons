# 27 — Чиби-арт пайплайн и аура редкости

**Фаза:** 7 — Арт и полировка · **Зависит от:** играбельная петля (00–14)

## Цель

Заменить плейсхолдеры на милый чиби-арт. Главное правило соло-разработки: **сначала
играбельность на квадратах, потом арт**. Производство — через один шаблон тела +
перекраска/аксессуары, а не 57 уникальных моделей.

## Стиль (GDD «Визуальный стиль»)

- Округлые большеголовые крупноглазые чиби; голова ≈40% тела, короткие лапки.
- Палитра по стихиям + мягкие тени, лёгкое свечение по редкости.
- Аура редкости: обычные — без эффектов; редкие+ — частицы; мифические — переливы;
  древние — золотая аура.

## Что делать

1. **Атлас спрайтов.** Texture atlas (Phaser) для драконов/растений/ресурсов/зданий.
   Загрузка в PreloadScene. Ключи спрайтов = `sprite`-поля из JSON (`dragons/embrix`).
2. **Шаблон тела + вариации.** Базовый чиби-силуэт, перекраска по стихии,
   аксессуары/детали по виду (рога, гребень, хвост — из строк 🎨 Бестиария). Это
   кардинально экономит время на 57 видах.
3. **Аура редкости как слой.** Отдельный шейдер/спрайт-оверлей, подставляется по
   `rarity` — единая система на всех драконов (частицы/переливы/золото).
4. **Скины гнёзд.** Уникальное гнездо на вид (строка 🏠 Бестиария) — атлас гнёзд,
   подстановка по `dragon.id` (task-18).
5. **Биомы, ресурсы, здания, остров, карта.** По описаниям 🎨 из GDD (биомы, иконки
   ресурсов, силуэты зданий, формы островов, пергаментная карта).
6. **Пайплайн ассетов.** Папка `art/src` (исходники) → экспорт в `public/assets` →
   упаковка в атлас (TexturePacker/free-tex-packer). Документировать в `docs/art.md`.
7. **Плейсхолдер-режим.** Сохранить возможность быстро переключаться на цветные
   квадраты (для прототипа новых видов до отрисовки).

## Definition of Done

- [ ] Драконы/растения/здания используют атлас, ключи берутся из JSON.
- [ ] Аура редкости — единый переиспользуемый слой по `rarity`.
- [ ] Гнёзда имеют скин по виду дракона.
- [ ] Добавление арта нового вида не требует правки кода (только атлас + sprite-ключ).
